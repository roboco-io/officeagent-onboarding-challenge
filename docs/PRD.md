# OfficeAgent BE 채용 과제 — Document Q&A API

## 과제 개요

문서를 업로드하면 내용을 분석하고, 사용자 질문에 대해 **문서 근거 기반 답변**을 생성하는 REST API 서버를 구현하세요.

---

## 핵심 요구사항

### 1. 문서 수집 (Ingestion)

- 문서 파일(PDF, TXT 등)을 업로드하면 텍스트를 추출하고, 검색 가능한 형태로 저장한다.
- 최소 2개 이상의 파일 포맷을 지원한다.

### 2. 질의응답 (Query & Answer)

- 사용자 질문에 대해 업로드된 문서에서 관련 내용을 검색(RAG)하고, LLM을 호출하여 답변을 생성한다.
- 답변에는 **출처(어떤 문서의 어떤 부분에서 왔는지)**가 포함되어야 한다.
- 문서에 근거가 없는 질문에 대해서는 환각 없이 적절히 처리해야 한다.

### 3. LLM 응답 캐싱

- 동일하거나 유사한 질문에 대해 LLM을 재호출하지 않도록 **캐시 DB**를 활용한다.
- 캐시 히트 여부를 응답에서 확인할 수 있어야 한다.
- 문서가 변경되면 관련 캐시가 무효화되어야 한다.

### 4. 프롬프트 설계

- RAG 파이프라인에 사용되는 프롬프트를 직접 설계한다.
- 프롬프트 설계 의도와 전략을 설명하는 문서(`PROMPT_DESIGN.md`)를 작성한다.

### LLM 연동 방식

별도의 API 키 발급 없이, **구독 기반으로 이용 가능한 SDK**를 사용합니다.

| SDK | 설치 | 비고 |
|-----|------|------|
| **Claude Code SDK** (`claude-code-sdk`) | `pip install claude-code-sdk` | Anthropic Claude 모델 사용. Claude Pro/Max 구독으로 이용 |
| **Codex CLI** (`@openai/codex`) | `npm install -g @openai/codex` | OpenAI 모델 사용. ChatGPT Pro 구독으로 이용 |

- 두 SDK 모두 로컬 CLI 도구를 프로그래밍 방식으로 호출하는 형태입니다.
- 어떤 SDK를 선택했는지, 서버 코드에서 어떻게 통합했는지를 `ARCHITECTURE.md`에 설명해주세요.
- 임베딩 모델은 선택한 SDK 또는 오픈소스 임베딩 모델(sentence-transformers 등)을 자유롭게 활용하세요.

벡터 DB, 캐시 DB, 청킹 전략, 검색 방식 등 **그 외 기술 선택은 지원자의 판단에 맡깁니다.**
선택의 이유를 `README.md` 또는 `ARCHITECTURE.md`에 간략히 설명해주세요.

## 평가 기준

| 영역 | 비중 | 주요 관점 |
|------|------|----------|
| **BE 설계** | 20% | 코드 구조, 레이어 분리, 비동기 처리, 에러 핸들링 |
| **RAG 파이프라인** | 25% | 청킹·임베딩·검색·답변 생성의 전체 흐름과 품질 |
| **LLM API 활용** | 20% | 호출 설계, 스트리밍, 구조화된 출력 |
| **캐싱 전략** | 20% | 캐시 설계, 유사 질문 처리, 무효화 전략 |
| **프롬프트 엔지니어링** | 15% | 프롬프트 품질, 환각 억제, 설계 문서 |

**완성도보다 설계 의도와 구현 품질을 우선 평가합니다.** 부분 구현도 평가 대상이며, 미구현 부분은 설계 문서로 대체할 수 있습니다.
