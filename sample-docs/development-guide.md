# 개발 가이드

## 코드 리뷰 정책

모든 PR은 최소 1명의 리뷰어 승인을 받아야 머지할 수 있습니다.
리뷰어는 PR 생성 후 24시간 이내에 첫 리뷰를 완료해야 합니다.

### 리뷰 체크리스트
- 기능이 요구사항을 충족하는가
- 테스트 코드가 포함되어 있는가
- 보안 취약점이 없는가
- 코드 스타일 가이드를 따르는가

## 브랜치 전략

Git Flow를 기반으로 합니다.

- `main`: 프로덕션 배포 브랜치
- `develop`: 개발 통합 브랜치
- `feature/*`: 기능 개발
- `hotfix/*`: 긴급 버그 수정
- `release/*`: 릴리즈 준비

## 배포 프로세스

1. `feature` → `develop` PR 머지
2. QA 환경 자동 배포 (CI/CD)
3. QA 팀 검증 완료 후 `release` 브랜치 생성
4. `release` → `main` 머지 후 프로덕션 배포
5. 프로덕션 배포 후 모니터링 (최소 1시간)

## 장애 대응

- P1 (서비스 전체 장애): 즉시 대응, 30분 내 원인 파악
- P2 (주요 기능 장애): 1시간 내 대응
- P3 (부분 기능 장애): 당일 내 대응
- P4 (경미한 버그): 다음 스프린트에 포함
