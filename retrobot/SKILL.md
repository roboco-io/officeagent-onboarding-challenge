# Retrobot — 자동 회고 생성 스킬

> 이 문서는 AI 에이전트(Claude Code / Codex)가 읽고 실행하는 지침입니다.

## 목적

프로젝트의 AI 에이전트 작업 로그를 분석하여 **KPT(Keep/Problem/Try) 회고**를 자동 생성한다.
매 push마다 자동으로 실행되어, 성장의 흐름을 타임라인으로 기록한다.

## 실행 절차

### 1단계: 로그 탐색

현재 프로젝트의 AI 에이전트 작업 로그를 찾는다.

- **Claude Code:** `~/.claude/projects/` 아래에서 현재 프로젝트 경로(`git rev-parse --show-toplevel`)에 매핑되는 디렉토리를 찾는다. 경로 구분자 `/`가 `-`로 인코딩되어 있다. (예: `/Users/foo/my-project` → `-Users-foo-my-project`)
- **Codex:** `~/.codex/` 아래 유사한 구조를 탐색한다.
- 로그 파일은 `*.jsonl` 형식이다.

로그를 찾지 못하면 "로그를 찾을 수 없습니다"라고 출력하고 종료한다.

### 2단계: 세션 분석

JSONL 로그를 읽고 다음을 추출한다:

- **사용자 프롬프트**: `type: "human"` 엔트리의 `message.content`
- **수행한 태스크**: 사용자 프롬프트 중 실질적인 작업 요청 (시스템 메시지, 태그 제외)
- **사용한 도구**: `type: "tool_use"` 또는 assistant 메시지 내 tool_use 블록
- **발생한 에러**: tool_result 중 error/failed/traceback 키워드 포함
- **세션 시간**: 각 세션의 시작/종료 타임스탬프

### 3단계: KPT 회고 작성

분석 결과를 바탕으로 아래 형식의 마크다운을 생성한다:

```markdown
# Retrobot 회고 — {날짜} {시간} KST

> 자동 생성된 회고입니다. 직접 편집하여 보완하세요.

## 요약

| 항목 | 값 |
|------|-----|
| 세션 수 | N |
| 총 프롬프트 | N |
| 에러 발생 | N |

## 세션별 활동

### 세션 `{id}` ({시작} ~ {종료})

**프롬프트:**
1. 실제 사용한 프롬프트 내용
2. ...

**발생한 에러:** (있는 경우)
- 에러 내용

## KPT 회고

### Keep (좋았던 것)
- 로그에서 파악한 긍정적 패턴 (에러 없이 완료, 효율적 도구 활용 등)

### Problem (나빴던 것)
- 반복된 에러, 비효율적 패턴, 실패한 시도 등

### Try (개선해야 할 것)
- Problem에 대한 구체적 개선 방안

## 타임라인

| 시각 | 세션 | 주요 활동 |
|------|------|----------|
| ... | ... | ... |
```

#### KPT 작성 가이드라인

- **Keep**: 단순 사실 나열이 아닌, "왜 좋았는지" 이유를 포함한다.
- **Problem**: 에러뿐 아니라 비효율(같은 작업 반복, 불필요한 시행착오)도 포함한다.
- **Try**: Problem 각각에 대응하는 구체적 액션 아이템을 제시한다.
- `<!-- 직접 추가하세요 -->` 플레이스홀더를 남겨 수동 보완을 유도한다.

### 4단계: 파일 저장

- 파일명: `retros/{YYYY-MM-DD}_{HHMM}_retro.md`
- `retros/TIMELINE.md` 테이블에 새 회고 링크를 추가한다.
- `retros/` 디렉토리가 없으면 생성한다.

### 5단계: Git 스테이징

- 생성된 `retros/` 내 파일을 `git add`한다.
- 커밋 메시지: `docs(retro): 자동 회고 생성 by Retrobot`

## 옵션

| 인자 | 설명 | 기본값 |
|------|------|--------|
| `--all` | 전체 세션 분석 | 최근 24시간만 |
| `--since {N}h` 또는 `{N}d` | 지정 기간 분석 | 24h |
